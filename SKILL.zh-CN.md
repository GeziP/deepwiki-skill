# DeepWiki 技术文档生成器

## Overview

**DeepWiki 是一种技术文档格式规范，强调架构可视化、API 完整性和代码示例可运行性。**

核心原则：
- 每个模块一份独立文档
- ASCII 图表代替 UML（轻量、可维护）
- API 参考表（签名 + 说明）
- 可运行代码示例
- 测试覆盖追踪
- 状态机与流程图可视化

**输出路径**: `doc/tech-docs/<ModuleName>_Design.md`

---

## Generation Process (四阶段)

必须完成每个阶段才能进入下一阶段。

### Phase 1: 源码分析

**生成任何内容之前:**

1. **读取目标文件**
   - 解析类/结构体定义
   - 提取公开 API（方法、函数）
   - 识别内部实现细节
   - 查找状态变量和枚举
   - 记录依赖关系（imports, includes）

2. **识别模块类型**
   - 有状态模块（有生命周期状态）→ 需要状态机图
   - 流程模块（有工作流）→ 需要流程图
   - 数据模块（结构体、枚举）→ 侧重数据结构
   - 工具模块（辅助函数）→ 侧重 API 参考

3. **检查相关文件**
   - 测试文件: `tests/<module>_test.cpp`, `test_<module>.py`
   - 配置文件: `<module>_config.h`, `<module>.yaml`
   - 相关模块: imports/dependencies

### Phase 2: 上下文收集

**收集支持信息:**

1. **查找现有文档**
   - 检查 `doc/tech-docs/` 相关文档
   - 记录版本历史、相关模块
   - 提取交叉引用链接

2. **提取测试覆盖**
   - 从测试文件列出测试用例名称
   - 识别已覆盖功能
   - 标注缺失测试区域

3. **检查 Git 历史**
   ```bash
   git log --oneline --follow <file> | head -10
   ```
   - 最近变更
   - 作者信息
   - 变更原因

### Phase 3: 文档生成

**使用 template.md 结构（12 章节）:**

| 章节 | 内容 | 必须 |
|------|------|------|
| 文档信息 | 版本、日期、读者、关联文档、源文件 | ✓ |
| 1. 概述 | 背景、问题、解决方案（含 ASCII 架构图） | ✓ |
| 2. 设计目标 | 功能目标、非功能目标 | ✓ |
| 3. 架构设计 | 模块关系图、职责划分表 | ✓ |
| 4. 核心概念 | 数据结构、枚举定义、常量定义 | ✓ |
| 5. 状态机 | 状态定义、转换图、转换条件 | ○ |
| 6. 流程图 | 核心流程图、步骤说明、异常处理 | ○ |
| 7. 实现细节 | 关键代码片段（带注释） | ✓ |
| 8. API 参考 | 公开接口表（签名、说明） | ✓ |
| 9. 使用指南 | 完整可运行代码示例 | ✓ |
| 10. 常见问题 | FAQ、陷阱、最佳实践 | ○ |
| 11. 测试覆盖 | 测试用例表、覆盖范围 | ✓ |
| 附录 | 文件清单、依赖关系、变更历史 | ○ |

(✓ = 必须, ○ = 可选 - 根据模块类型决定)

### Phase 4: 质量验证

**输出最终文档前:**

1. **检查 ASCII 图表**
   - 所有框正确闭合
   - 箭头方向正确
   - 标签可读

2. **验证代码示例**
   - 示例完整（非片段）
   - 注释解释 WHY，不只是 WHAT
   - 包含 import/include 语句

3. **交叉引用链接**
   - 使用 `[Module_Design.md]` 链接相关文档
   - 使用相对路径链接源文件
   - 链接测试文件

4. **最终审查**
   - 文档遵循 template.md 结构
   - 所有必须章节存在
   - 表格格式正确

## ASCII 图表风格

### 架构图

```
┌─────────────────────────────────────────────────────────────┐
│                    模块架构                                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────┐     ┌─────────────┐                       │
│  │  ComponentA │────▶│  ComponentB │                       │
│  └─────────────┘     └─────────────┘                       │
│         │                   │                               │
│         ▼                   ▼                               │
│  ┌─────────────┐     ┌─────────────┐                       │
│  │  ComponentC │     │  ComponentD │                       │
│  └─────────────┘     └─────────────┘                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**元素**：
- `────▶` 数据流/调用方向
- `│` `▼` 层级关系
- `┌─┐` `└─┘` 边框

### 状态机图

```
                    ┌─────────────┐
                    │   Initial   │
                    └─────────────┘
                          │
                    start()
                          │
                          ▼
┌─────────────┐     ┌─────────────┐
│   Failed    │◀────│   Running   │
└─────────────┘ fail└─────────────┘
                          │
                    succeed
                          │
                          ▼
                   ┌─────────────┐
                   │  Completed  │
                   └─────────────┘
```

**元素**：
- `───▶` 状态转换方向
- `◀───` 反向转换
- 转换条件标注在箭头旁

### 流程图

```
┌─────────┐
│  Start  │
└────┬────┘
     │
     ▼
┌─────────┐     ┌─────────┐
│ Step 1  │────▶│ Step 2  │
└─────────┘     └────┬────┘
                     │
                ┌────┴────┐
                ▼         ▼
           ┌─────────┐ ┌─────────┐
           │  Yes    │ │   No    │
           └────┬────┘ └────┬────┘
                │           │
                └─────┬─────┘
                      ▼
               ┌─────────┐
               │   End   │
               └─────────┘
```

**元素**：
- `┌───┐` 矩形框表示步骤
- `◀───▶` 分支/合并
- `Yes/No` 条件分支标注

## 代码示例风格

```cpp
// ========== 基本使用 ==========

// 创建任务
auto task = std::make_shared<Task>(Task::Config{
    .name = "SampleTask",
    .priority = 10
});

// 执行
task->execute();

// ========== 进阶用法 ==========

// 带回调
task->setOnSuccess([](const TaskHookContext& ctx) {
    std::cout << "Task " << ctx.taskName << " completed\n";
});
```

**要求**：
- `// ========== 标题 ==========` 分节
- 注释解释 WHY（为什么这样做）
- 示例必须可运行（包含必要的 import/include）

## Template 参考

查看 `template.md` 了解完整的章节结构和占位符：

| 占位符 | 说明 |
|--------|------|
| `{{MODULE_NAME}}` | 模块名称（如 Task, Module） |

| `{{DATE}}` | 当前日期 |

| `{{TEST_FILE}}` | 测试文件路径 |

## Red Flags - 停止并重新分析

如果遇到：
- 找不到源文件 → 询问用户正确路径
- 单文件多类 → 询问要文档哪个，或建议拆分
- 无测试文件 → 在文档中标注，建议创建测试
- 复杂依赖图 → 侧重直接依赖，间接依赖放附录

**禁止：**
- 不读源码猜测模块功能
- 不理解关系就画图
- 复制模板不填充实际内容
- 内容少就跳过必须章节
